# 개인정보 유출 이후 이커머스 부정결제 탐지 FDS

> Groom Fintech 6기 프로젝트, 4조

대규모 개인정보 유출로 크리덴셜이 탈취된 상황을 전제로, 로그인과 결제 두 단계에서 계정 탈취(ATO)와 부정결제를 탐지하는 FDS를 기획하고 공개 데이터로 검증했습니다.

---

## 문제 배경

2026년 5월 티빙 개인정보 유출로 1,953만 명의 아이디, 비밀번호, CI, DI가 유출됐습니다. 티빙은 사고 직후 CJ ONE 연동 계정을 전면 잠갔는데, 이는 정상 사용자 전원을 위험으로 판정한 **오탐률 100%의 조치**였습니다.

우리는 여기서 질문을 시작했습니다. **전면 잠금 대신, 위험한 로그인과 결제만 골라낼 수는 없었을까?**

---

## 접근 방법: 2단 방어선

공격은 두 단계로 진행되므로, 방어도 두 단계로 설계했습니다.

```
개인정보 유출 (외부 서비스)
        ↓
[1단] 로그인 방어  ← 크리덴셜 스터핑, 계정 탈취 탐지
        ↓
[2단] 결제 방어    ← 등록 결제수단 부정 사용 탐지
```

| 단계 | 탐지 대상 | 사용 데이터 | 핵심 신호 |
| :--- | :--- | :--- | :--- |
| 1단 (로그인) | 크리덴셜 스터핑, ATO | RBA Login Dataset | 같은 IP의 다계정 시도, 낯선 국가, 낯선 기기 |
| 2단 (결제) | 부정결제 | IEEE-CIS Fraud Detection | 금액 이상, 기기 변경, 이메일 도메인 불일치 |

두 데이터는 서로 다른 서비스라 계정 키로 직접 조인되지 않습니다. 공개 데이터의 한계이며, **위험 점수를 이어주는 파이프라인**으로 시나리오를 완성했습니다.

---

## 검증 결과

**1단 로그인 탐지 (RBA, 50만 건 샘플)**

- 크리덴셜 스터핑 신호 확인: 공격 IP의 계정 수 중앙값 7개 (정상 2개, 최대 268개)
- 룰 `ip_user_count >= 10`: 공격의 59%를 정밀도 0.28로 포착 (기저 9.5% 대비 약 3배)
- 지도학습(GBDT): ROC-AUC 0.86, PR-AUC 0.56
- 이상탐지(Isolation Forest)는 약했음. 공격이 희소 이상치가 아니라 다수 패턴이기 때문

**2단 결제 탐지 (IEEE-CIS, 59만 건)**

- 사기율 3.5%, 정상이 사기보다 약 28배 많은 극단적 불균형
- 지도학습(LightGBM): AUC 0.826, Recall 67% (기준선 0.5)
- 기준선 조정 시 Recall 85~93%까지 상승. "미탐이 더 치명적"이라는 설계에 따라 기준선을 낮춤
- SHAP 분석: 카드 나이, 결제 금액, 고객 거래 횟수가 상위 신호

---

## 핵심 설계 관점

**미탐이 오탐보다 치명적입니다.** 승인 즉시 피해가 확정되고(특히 상품권은 즉시 사용되어 회수 불가), 되돌릴 수 없기 때문입니다. 따라서 Recall을 우선합니다.

**다만 차단이 답은 아닙니다.** 오탐을 차단이 아닌 **추가 인증**으로 처리해 정상 고객은 본인 확인 한 번으로 통과하게 합니다. 가드레일로 "정상 결제 중 인증이나 차단을 겪은 비율 2% 이하"를 두고, 이 선을 넘으면 정책을 되돌립니다.
