# PRD - 이다영

## 0. 요약
개인정보 유출로 크리덴셜이 도용된 **이커머스 간편결제 이용자**가, 낯선 기기·위치에서 시도되는 **상품권 등 환금성 부정결제가 승인되기 전에** 단계별 인증으로 걸러져 **자산과 계정 통제권을 지키도록** 한다. 정상 이용자는 평소처럼 결제한다.

## 1. 문제 (Problem)

- 2026년 5월 티빙에서 **1,953만 명**의 아이디·비밀번호·CI/DI·환불계좌가 유출됐다. 유출된 크리덴셜은 다른 서비스에 대입되는 **크리덴셜 스터핑**으로 이어진다. (CJ ONE은 2차 피해를 막으려 연동 계정을 전면 잠금)
- 이커머스 간편결제 계정에는 **카드·계좌가 이미 등록**돼 있어, 계정이 뚫리는 순간 결제수단이 통째로 노출된다.
- 피해자는 **결제가 승인된 뒤에야 안다.** 공격자가 이메일·비밀번호를 먼저 바꾸면 알림조차 받지 못한다.
- 특히 **상품권·기프트카드는 승인 즉시 현금화**돼 회수가 불가능하다. 차지백 확정까지는 2~8주가 걸린다.
- 기존 대안인 **전면 계정 잠금은 오탐이 과도**하다. 실제 로그인 데이터 검증 결과 "낯선 국가 + 낯선 기기" 조건은 계정탈취 11건을 모두 잡았지만 **경보가 13만 건**(정밀도 0.00008) 발생했다.

## 2. 목표 (Goal)

- 도용된 결제를 **승인 전에** 멈춘다. (승인 후에는 회수 불가)
- 정상 이용자는 **평소처럼 결제**하게 한다. 오탐 비용은 차단이 아닌 **추가 인증 단계로 흡수**한다.
- 제품 포지션: **"차단하는 FDS"가 아니라 "한 번 확인하는 FDS"**
- 회사 관점: 부정결제 피해액·차지백 비용을 줄이고, 유출 사고 국면에서도 결제 신뢰를 유지한다.

## 3. 사용자 (User)

- **김지현 (28세, 직장인, 서울)** — 간편결제에 카드 3장을 등록해두고 주 2~3회 모바일 결제. 대부분 5만 원 이하.
- 가입 서비스가 20개 이상이고, 상당수에 **같은 비밀번호를 재사용**한다.
- 티빙 유출 안내 문자를 받았지만 어느 사이트에 같은 비밀번호를 썼는지 기억하지 못했다.
- **Pain:** 내 계정이 이미 뚫렸는지 알 방법이 없다. 알림을 놓치면 피해가 그대로 확정된다.
- **Need:** 내가 아닐 때만 막아주고, **나일 때는 방해받지 않기.** 막혔다면 이유를 알고 싶다.
- **첫 고객 선정 이유:** 결제수단이 이미 등록돼 피해가 즉시 발생하고(시급성), 비밀번호 재사용으로 공격에 노출돼 있으며(적합성), 모수가 가장 크다(확장성).

## 4. 핵심 기능 (Features)

- **① 세션 리스크 스코어링** — 로그인 시점에 신규 기기·신규 국가·동일 IP 다계정 접속·로그인 실패 횟수를 집계해 세션 위험 점수를 부여한다. → 결제 전에 도용을 감지
- **② 결제 시점 위험 판정** — 세션 위험 점수에 결제 신호(고환금성 카테고리, 평소 대비 금액 이탈, 발신·수신 이메일 도메인 불일치)를 결합해 최종 점수를 낸다. → 승인 순간에 막아야 회수 가능
- **③ 위험 수준별 단계 조치** — 자동 승인 → 추가 인증(OTP) → 결제 보류 + **변경 전 원래 연락처로 알림** → 차단 + 계정 보호 잠금. → 차단 일변도의 오탐 비용을 인증으로 흡수
- **④ 운영 검토 큐** — 위험 점수 상위 K건을 **걸린 이유**와 함께 제공한다. → 제한된 인력으로 판단 가능

> Out of Scope: 전 계정 일괄 잠금(가설과 정반대), GPS 정밀 위치 수집(위치정보법상 별도 동의·사업신고 부담 → IP 기반으로 한정), 자체 결제 실행

## 5. 성공 지표 (Metric)

- **북극성 지표: 월간 부정결제 확정 피해 금액(원)** — 차지백·신고로 확정된 피해액 합계, 이벤트 후 **30일 확정 기준**(라벨 지연 반영)
- **보조 지표:** 승인 전 차단율, Recall(미탐 최소화가 1순위), Precision@K(검토 인력 기준)
- **가드레일 지표(넘으면 정책 롤백)**

| 지표 | 한계선 |
| :--- | :--- |
| 정상 결제 중 추가 인증·보류·차단을 겪은 비율 | ≤ 2.0% |
| 운영팀 일일 경보량 | ≤ 500건 |
| P95 조치 시간 | ≤ 30분 |

- **Accuracy는 쓰지 않는다.** 사기가 3.5%뿐이라 전부 정상이라고 예측해도 96.5%가 나오기 때문이다.
